# Пользовательские сценарии для проекта StudyBalance

---

## Сценарий 1 — Регистрация и вход в аккаунт
* Пользователь открывает сайт StudyBalance в браузере.
* На экране отображается форма регистрации или входа.
* Пользователь вводит логин и пароль.
* При регистрации пользователь вводит электронную почту.
* Система проверяет корректность введённых данных.
* При ошибке отображается сообщение с описанием проблемы.
* После успешного ввода пользователь отправляет форму.
* Система создаёт аккаунт или выполняет вход.
* Пользователь автоматически попадает на страницу «Сегодня».

---

## Сценарий 2 — Настройка учебного профиля
* После входа система предлагает выбрать учебный профиль.
* Пользователь выбирает подходящий вариант (например, 10–11 класс).
* Система отображает список доступных предметов.
* Пользователь отмечает изучаемые предметы.
* Пользователь может добавить собственный предмет.
* Пользователь сохраняет выбранный набор предметов.
* Система создаёт список предметов пользователя.
* Пользователь переходит к работе с задачами.

---

## Сценарий 3 — Создание кастомного профиля
* Пользователь выбирает вариант «другое» при выборе профиля.
* Система отображает список возможных параметров профиля.
* Пользователь выбирает нужные функции и типы задач.
* Пользователь задаёт собственный список предметов.
* Система отображает предварительный результат настройки.
* Пользователь подтверждает выбранные параметры.
* Система сохраняет профиль пользователя.
* Пользователь переходит к работе с задачами.

---

## Сценарий 4 — Добавление учебной задачи
* Пользователь нажимает кнопку добавления задачи (плюс в правом нижнем углу).
* Открывается форма создания задачи.
* Пользователь вводит название задачи.
* Пользователь выбирает предмет и тип задания.
* Пользователь задаёт дедлайн и сложность задачи.
* Пользователь указывает приоритет задачи (низкий, средний, высокий).
* Пользователь сохраняет задачу.
* Система добавляет задачу в список и отображает её в календаре.

---

## Сценарий 5 — Просмотр и управление задачами на неделе
* Пользователь открывает вкладку «Неделя».
* Система отображает задачи по дням недели.
* Пользователь просматривает распределение нагрузки.
* Пользователь выбирает конкретный день.
* Система показывает список задач выбранного дня.
* Пользователь может открыть задачу для редактирования.
* Пользователь переносит задачу на другой день.
* Система обновляет отображение задач в неделе.

---

## Сценарий 6 — Работа со страницей «Сегодня»
* Пользователь открывает страницу «Сегодня».
* Система отображает задачи на текущий день.
* Пользователь видит уровень общей нагрузки.
* Пользователь открывает задачу из списка.
* Пользователь отмечает задачу выполненной.
* Система переносит задачу в архив выполненных.
* Пользователь может перенести оставшиеся задачи на другой день.
* Система обновляет список задач текущего дня.

---

## Сценарий 7 — Просмотр статистики
* Пользователь открывает вкладку «Статистика».
* Система отображает графики нагрузки по дням недели.
* Пользователь выбирает интересующий график.
* Система показывает подробные данные по выбранному периоду.
* Пользователь переключается между типами графиков (по предметам, по дням).
* Пользователь анализирует распределение нагрузки.
* Пользователь делает вывод о перегруженных днях.
* Пользователь возвращается к планированию задач.

---

## Сценарий 8 — Настройка уведомлений
* Пользователь открывает настройки приложения.
* Переходит в раздел уведомлений.
* Выбирает типы уведомлений (дедлайны, напоминания).
* Устанавливает время получения уведомлений.
* При необходимости подтверждает электронную почту.
* Пользователь сохраняет настройки.
* Система применяет выбранные параметры.
* Пользователь возвращается к работе с задачами.

---

## Сценарий 9 — Резервное копирование и восстановление
* Пользователь открывает раздел резервного копирования.
* Выбирает создание резервной копии.
* Система формирует файл с данными пользователя.
* Пользователь скачивает файл на устройство.
* Пользователь открывает раздел восстановления.
* Выбирает файл резервной копии.
* Подтверждает восстановление данных.
* Система загружает данные и обновляет список задач.

---

## Сценарий 10 — Работа нескольких пользователей
* Пользователь открывает сайт StudyBalance.
* Вводит логин и пароль.
* Система выполняет авторизацию.
* Пользователь попадает в свой личный кабинет.
* Пользователь создаёт и редактирует свои задачи.
* Пользователь просматривает свои предметы и статистику.
* Пользователь выходит из аккаунта.
* Другой пользователь может войти в систему под своими данными.
