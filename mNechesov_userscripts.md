# Пользовательские сценарии для проекта StudyBalance

---

### Сценарий 1 — Регистрация и вход в аккаунт

* Пользователь открывает сайт StudyBalance в браузере.
* На экране появляется форма регистрации: логин, пароль, электронная почта.
* Пользователь вводит логин.
* Если логин уже существует, система выводит сообщение:  
  «Такой логин уже используется. Пожалуйста, выберите другой.»
* Пользователь вводит пароль.
* Если пароль короче 6 символов, система выводит сообщение:  
  «Пароль должен содержать от 6 до 30 символов.»
* Пользователь вводит адрес электронной почты.
* Если формат неверный, система сообщает об ошибке.
* Система отправляет письмо с подтверждением регистрации.
* После подтверждения пользователь автоматически входит в систему.

---

### Сценарий 2 — Настройка учебного профиля

* Пользователю предлагается выбрать учебный профиль (7–9 класс, 10–11 класс, колледж, другое).
* При выборе пункта «другое» запускается сценарий создания кастомного профиля.
* Система предлагает ознакомиться с особенностями выбранного профиля.
* Пользователь подтверждает или пропускает ознакомление.
* Далее открывается список предметов данного профиля.
* Пользователь отмечает изучаемые предметы.
* После сохранения запускается сценарий добавления задач.

---

### Сценарий 3 — Создание кастомного профиля

* Пользователь открывает раздел создания учебного профиля.
* Отображается список доступных функций профиля.
* Напротив каждой функции расположен чекбокс.
* Пользователь выбирает необходимые функции.
* Система отображает итоговую конфигурацию профиля.
* Пользователь может:
  * подтвердить настройки профиля
  * вернуться назад и изменить параметры.
* После подтверждения профиль сохраняется и становится активным.

---

### Сценарий 4 — Добавление учебной задачи

* Пользователь открывает раздел **«Задачи»**.
* Нажимает кнопку **«Добавить задачу»**.
* Открывается форма ввода параметров задачи.
* Пользователь заполняет:
  * название задачи
  * предмет
  * тип задания
  * срок выполнения
  * уровень сложности.
* При ошибках система выводит предупреждающие сообщения.
* После сохранения задача появляется в списке задач и календаре недели.
* Пользователь может:
  * добавить новую задачу
  * перейти к просмотру недели
  * вернуться на страницу «Сегодня».

---

### Сценарий 5 — Просмотр недельного расписания

* Пользователь открывает вкладку **«Неделя»**.
* Приложение отображает задачи по дням недели.
* Загруженные дни подсвечиваются цветом.
* Пользователь может:
  * открыть задачу
  * изменить параметры задачи
  * перенести задачу на другой день.
* Если выбранный день становится перегруженным, система выводит предупреждение.
* После изменения пользователь может продолжить редактирование расписания или перейти в другой раздел.

---

### Сценарий 6 — Страница «Сегодня»

* Пользователь открывает страницу **«Сегодня»**.
* Отображаются все задачи, запланированные на текущий день.
* Пользователь видит общий уровень нагрузки.
* Пользователь может:
  * открыть задачу
  * отметить задачу выполненной
  * перенести задачу на другой день.
* После выполнения задачи пользователь отмечает её завершённой.
* Выполненные задачи перемещаются в архив.
* Пользователь может перейти к просмотру статистики или недельного расписания.

---

### Сценарий 7 — Работа со статистикой

* Пользователь переходит во вкладку **«Статистика»**.
* Приложение анализирует накопленные данные.
* Отображаются графики нагрузки и выполнения задач.
* Пользователь выбирает тип визуализации.
* При наведении отображаются подробные данные.
* Возможен экспорт графиков и таблиц.
* Данные используются для построения рекомендаций.

---

### Сценарий 8 — Напоминания и уведомления

* Пользователь открывает настройки уведомлений.
* Выбирает типы напоминаний.
* Устанавливает время получения уведомлений.
* Система проверяет подтверждение электронной почты.
* При необходимости запрашивается подтверждение email.
* Напоминания сохраняются в профиле пользователя.
* После сохранения пользователь продолжает работу с приложением.

---

### Сценарий 9 — Резервное копирование и восстановление данных

* Пользователь открывает раздел **«Резервное копирование»**.
* Выбирает действие **«Создать резервную копию»**.
* Система показывает информацию о данных, которые будут сохранены.
* Пользователь подтверждает создание копии.
* Формируется файл резервной копии.
* Пользователь может:
  * скачать файл
  * сохранить его в облачное хранилище.

**Восстановление данных**

* Пользователь открывает раздел **«Восстановление данных»**.
* Загружает файл резервной копии.
* Система предупреждает о замене текущих данных.
* Пользователь подтверждает восстановление.
* После завершения восстановления пользователь проверяет задачи, расписание и статистику.

---

### Сценарий 10 — Работа нескольких пользователей

* Приложение развёрнуто на сервере.
* Пользователи регистрируются и входят в систему под своими аккаунтами.
* Каждый пользователь создаёт и редактирует собственные задачи.
* При открытии разделов **«Задачи»**, **«Неделя»** и **«Статистика»** пользователь видит только свои данные.
* Пользователи могут работать в системе одновременно.
* Работа одного пользователя не влияет на данные других пользователей.
