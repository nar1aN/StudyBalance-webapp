# **Пользовательские сценарии для проекта StudyBalance**

# **Сценарий 1 — Регистрация и вход в аккаунт**

1. Пользователь открывает сайт StudyBalance в браузере.
2. На экране появляется форма регистрации: логин, пароль, электронная почта.
3. Пользователь вводит логин.

   * Если логин уже существует, система выводит сообщение:
     *«Такой логин уже используется. Пожалуйста, выберите другой.»*
   * Если второй логин тоже занят, система даёт рекомендации:
     *«Добавьте цифру, используйте никнейм или имя-фамилию.»*
4. Пользователь вводит пароль.

   * Если пароль короче 6 символов, система выводит:
     *«Пароль должен содержать от 6 до 30 символов.»*
5. Пользователь вводит email.

   * Если формат неверный, появляется сообщение:
     *«Введите корректный адрес электронной почты.»*
6. Если все данные корректны, система отправляет письмо с подтверждением.
7. Пользователь переходит по ссылке из письма.
8. Система приветствует пользователя и предлагает выполнить первоначальную настройку.
9. Далее автоматически запускается Сценарий 2 — Настройка учебного профиля.

# **Сценарий 2 — Настройка учебного профиля**

1. Пользователю предлагается выбрать свой учебный профиль:
   а) 7–9 класс
   б) 10–11 класс
   с) колледж / лицеист
   д) другое
2. Если выбран пункт «другое», запускается Сценарий 3 — Создание кастомного профиля.
3. После выбора профиля система предлагает познакомиться с его особенностями.

   * Если пользователь нажимает «Да», отображается описание:

     * какие предметы включены;
     * какие типы задач доступны;
     * какие подсказки нагрузки предусмотрены.
   * Если пользователь выбирает «Нет», он может сразу перейти далее.
4. После подтверждения профиля пользователь переходит к выбору предметов.
5. Пользователь отмечает те предметы, которые изучает в текущем году.
6. После сохранения настроек запускается Сценарий 4 — Первое добавление задач.

# **Сценарий 3 — Создание кастомного профиля**

1. Пользователь видит список функций, которые можно включить вручную:

   * планирование задач;
   * визуализация нагрузки;
   * уведомления о дедлайнах;
   * дневник выполнения;
   * аналитика по предметам;
   * прогнозирование нагрузки на неделю.
2. Напротив каждой функции — чекбокс.
3. Пользователь отмечает те функции, которые ему нужны.
4. Параллельно на экране формируется итоговая конфигурация профиля.
5. После нажатия «Далее» система предупреждает:
   *«Вы не сможете изменить профиль в течение недели. Продолжить?»*
6. На следующем экране пользователь подтверждает выбор или возвращается назад.
7. После сохранения профиля StudyBalance завершает начальную настройку.
8. Пользователь переходит к добавлению своих первых задач.

# **Сценарий 4 — Первое добавление учебных задач**

1. Пользователю показывается пустой список задач с кнопкой «Добавить задачу».
2. При нажатии открывается форма:

   * название задачи;
   * предмет;
   * тип задачи (домашка / проект / подготовка к контрольной / олимпиада);
   * срок выполнения (дата);
   * примерная сложность (оценка нагрузки).
3. Пользователь заполняет поля.
4. Если срок установлен на прошедший день, система предупреждает.
5. После сохранения задача появляется в списке и в недельном планировщике.
6. Приложение автоматически пересчитывает нагрузку недели и подсвечивает дни по уровню занятости.
7. Пользователь может перейти к Сценарию 5 — Просмотр задач на неделю.

# **Сценарий 5 — Просмотр недельного расписания**

1. Пользователь открывает вкладку «Неделя».
2. Приложение отображает задачи по дням недели.
3. Дни с высокой нагрузкой подсвечиваются красным или оранжевым.
4. Пользователь может:

   * перенести задачу на другой день;
   * изменить срок;
   * добавить новую задачу;
   * отметить задачу как выполненную.
5. При переносе задачи приложение автоматически пересчитывает нагрузку и меняет подсветку.
6. Если пользователь создаёт слишком плотный день, система выводит предупреждение:
   *«У вас слишком много задач в этот день. Желательно перенести частично на другой день.»*

# **Сценарий 6 — Страница «Сегодня»**

1. На странице отображаются все задачи, запланированные на сегодня.
2. Приложение показывает общий объём нагрузки (в баллах).
3. Ниже появляются рекомендации:

   * «Можно перенести *математику* на завтра — там меньше нагрузки.»
   * «У вас три сложные задачи подряд — попробуйте распределить их.»
4. Пользователь может:

   * завершить задачу (чекбокс);
   * отложить задачу;
   * удалить задачу;
   * изменить сложность, если она оказалась другой.
5. После выполнения задачи она перемещается в список выполненных.

# **Сценарий 7 — Работа со статистикой**

1. Пользователь открывает вкладку «Статистика».
2. Приложение анализирует данные и показывает графики:

   * количество задач по предметам;
   * нагрузка по дням недели;
   * процент выполнения задач;
   * динамика нагрузки за месяц.
3. Пользователь может выбрать тип графика: диаграмма, линия, столбцы.
4. При наведении на график отображаются подробности.
5. Пользователь может сохранить графики в PNG или PDF.
6. Можно экспортировать статистику в CSV.

# **Сценарий 8 — Напоминания и уведомления**

1. Пользователь открывает раздел «Настройки уведомлений».
2. Он может включить напоминания:

   * за 1 день до дедлайна;
   * за 3 часа;
   * утром в день выполнения;
   * еженедельную сводку нагрузки.
3. После включения уведомлений StudyBalance отправляет email-напоминания.
4. Если email не подтверждён, система просит пройти подтверждение.

# **Сценарий 9 — Резервное копирование данных**

1. В разделе «Настройки» пользователь выбирает «Резервная копия».
2. Можно выбрать способ сохранения:

   * на локальный диск;
   * в облако;
   * экспорт SQLite-базы.
3. Система предупреждает:
   *«Резервная копия включает ваши задачи, предметы и статистику.»*
4. После подтверждения создаётся файл резервной копии.
5. Пользователь может восстановить данные из файла.

# **Сценарий 10 — Работа нескольких пользователей**

1. Приложение установлено на школьном или домашнем сервере.
2. Несколько учеников заходят под своими логинами.
3. Каждый видит только свои задачи и статистику.
4. Если одновременно несколько пользователей добавляют задачи, система корректно обновляет базу.
5. Одновременная работа до 5 пользователей проходит без задержек.
